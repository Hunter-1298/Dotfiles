### Frontend ↔ cdao_powerbi live call inventory (post-merge)

Only these `/api/v1/powerbi/*` paths have real (non-generated-SDK-only) callers post-merge:

- `GET /powerbi/reports`, `POST /powerbi/reports`, `PATCH /powerbi/reports/{slug}`, `DELETE /powerbi/reports/{slug}` — `routes/admin/power-bi.tsx` (hub.py)
- `GET /powerbi/embed-config/{slug}` — `PowerBIEmbed.tsx` + `LivePowerBIEmbed.tsx` (hub.py)
- `GET /powerbi/datasets/{id}/schema` — `useDatasetSchema` → ChartPicker (datasets.py)
- `POST /powerbi/charts/data` — `useChartData` → ChartWidget (charts.py)
- `GET/PUT /powerbi/workspaces/{id}/layout` — `useWorkspaceLayout` → `routes/dashboard.tsx` (workspace_layout.py)

Everything else exposed by `cdao_powerbi.routes` is referenced **only** through `lib/api/generated/*` and `lib/powerbi-api.ts` re-exports with no consumer in the post-merge tree.

3. Pruning the corresponding helpers in `apps/aca/frontend/src/lib/powerbi-api.ts` and any orphaned hook (`useWizardSchema`, `useDashboardThumbnail`, `usePowerBi`, `usePowerBiAdmin`) — these have no consumers either way.

## Dead `instance_id` branch in `dashboard_agent.py`

`apps/aca/backend/app/api/routes/dashboard_agent.py:71-90` runs only when
`body.instance_id is not None`. Post-PR-#178 the only caller of
`POST /api/v1/dashboards/agent/plan` is `DashboardAgentDock` mounted in
`routes/dashboard.tsx:291` with `instanceId={null}`, so the branch is
dead. The branch is **worth removing in the same follow-up PR** — it is
the only reason `dashboard_agent.py` imports `DashboardInstanceStorage`,
`UserDashboardPermissionStorage`, and `PermissionService`, and it pulls
in the only non-test caller of `cdao_powerbi.services.permission_service`
in the codebase. Removing it:

- simplifies the route to a single workspace-canvas path,
- drops the `cdao_powerbi.storage` + `cdao_powerbi.services.permission_service` imports from `dashboard_agent.py`,
- aligns with the planned `instances.py` deletion (the branch was the only consumer of `assert_at_least` outside the deleted `/dashboards/instances/$id/edit` page),
- `apps/aca/backend/tests/api/test_dashboard_agent_routes.py` will need its instance-mode test cases removed in the same change. The frontend type `DashboardAgentPlanRequest.instance_id` (`lib/dashboard-agent-api.ts:40`) and the `instanceId` prop on `DashboardAgentDock` / `DashboardAgentComposer` would also become dead — drop them or leave as `null`-only for forward compat (small call).
